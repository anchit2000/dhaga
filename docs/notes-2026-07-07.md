- `downscalePhoto`'s EXIF-orientation handling turned out to be a non-issue,

  `PhotoCropper` draws from an `<img>` element, which all evergreen browsers
  auto-rotate per EXIF before decode — so the cropped canvas output is always
  already correctly oriented with no EXIF tag left to misinterpret. Worth
  knowing this is a structural property of the pipeline order, not something
  `downscalePhoto` itself guards — if a future change ever calls
  `downscalePhoto` on a raw pre-crop file, the orientation bug would return.

  instead of throwing `OverconstrainedError`.

HEVC codec) — only Safari can, by delegating to the OS's licensed decoder.
iPhones default to HEIC for camera photos. So the "scan the card with your
phone, upload the file from your laptop" flow this app explicitly supports —
if that laptop isn't running Safari — silently broke: the image never loads,
`rect` never gets set, "Use photo" stays disabled forever, with zero
explanation. Fixed with a minimal `onError` → clear message + "choose another
photo" affordance (no HEIC transcoding library added — that'd be
over-engineering for this fix). Typecheck/lint/vitest all green.
