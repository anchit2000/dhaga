## The one real code gap I found (§3): embeddings cleanup for note deletion

- `forgetContact` (`apps/web/src/lib/repo/contacts/mutations.ts:68-95`) does a
  full, correct cascade including `deleteEmbeddingsByContact` — no gap there.
- But `deleteNote` (`apps/web/src/lib/repo/notes.ts:43-50`) only tombstones
  notes/facts/edges — it does **not** delete the note's embeddings itself.
  Embeddings cleanup (`deleteEmbeddingsForNote`) only happens one layer up,
  in the server action `deleteNoteAction` (`apps/web/src/lib/actions/notes.ts:56-64`).
- That means any other caller of `deleteNote` (e.g. `card-images.test.ts:50`
  calls it directly) silently leaves orphaned embedding rows. It also means
  the invariant "delete a note → its embeddings go with it" lives in the
  action layer, not the repo layer where the rest of the cascade logic lives
  — an easy place for a future caller to get it wrong.
- The "receipts cascade" test (`graph-receipts.test.ts`) never asserts
  anything about the `embeddings` table, so this gap is untested.

**I'm sending a subagent to fix this**: move the embeddings cleanup into
`deleteNote` itself (so it can't be forgotten by any caller), and add a test
asserting embeddings are gone after both `deleteNote` and `forgetContact`.
Small, self-contained, no live API keys needed. Will commit locally, not push.

## Update — 2026-07-11 (continued): signal-detection + heuristic-parse audit

Third algorithm-audit pass (after graph-traversal/decay-scoring and
name-cluster dedup earlier this session), targeting signal detection and the
non-LLM heuristic contact parser. Found and fixed three real bugs (`46b74f6`,
merging `c949b15`):

- **`heuristicContactParse` dropped names entirely for NFD-encoded Unicode**
  (e.g. macOS-originated text storing "é" as "e" + a combining acute mark) —
  `isNameLike`'s `\p{L}` regex doesn't match combining marks, so the whole
  line failed and the contact saved with no name. Fixed by NFC-normalizing
  input before parsing. Distinct from `normalizeForMatch` (NFD-strip, for
  accent-*insensitive* equality) — this fix preserves the accent, it just
  composes it into one codepoint first.
- **Same file: link dedup ran before punctuation-stripping**, so the same
  URL mentioned once mid-sentence and once at a sentence's end (trailing
  period swallowed into the regex match) survived as a visible duplicate.
  Fixed by stripping before deduping.
- **Nightly signal-detection sweep had no duplicate guard** — nothing
  auto-updates a contact's title/company when a job-change signal fires
  (turning it into a note is an explicit user action), so an unresolved
  signal looked "new" again every ~6-day rescan, flooding the Home feed with
  repeats of the same event. Added `hasOpenSignal` guard in `repo/signals.ts`.

Also audited (found correct, no change): `graph.ts`'s `applyExtraction`
receipts/cascade, `forgetContact`'s tombstone behavior, and note-extraction's
orchestration layer (no deterministic post-processing lives there to audit).
